# Contributing to Claude Learnings

## What to contribute

**✅ Contribute to `skill/SKILL.md`:**
- Better capture behaviour
- Better load/apply logic
- New commands
- Fixes to how learnings are organised or surfaced

**❌ Never contribute your personal learnings:**
- `by-domain/` files are yours — they contain your project history
- `by-project/` files are yours
- The `example-learnings/` folder exists only to show the format

## Contribution format for skill improvements

Open a PR with:
1. What behaviour was wrong / missing
2. The change to `SKILL.md`
3. An example of the corrected behaviour

## Adding a new command

If you add a new slash command (e.g. `/learnings export`), document it in `SKILL.md` following the same structure as existing commands.

## Rule format (for example-learnings only)

```markdown
### Short descriptive title
- ❌ What not to do — specific, not vague
- ✅ What to do instead — specific, actionable
- 📁 Project: <name> | Date: <YYYY-MM>
```

One rule = two lines max. No essays.
